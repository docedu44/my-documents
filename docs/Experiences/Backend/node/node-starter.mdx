## Install
برای نصب کافیست ک وارد سایت
[Node js](https://nodejs.org/en/)
شویم و بر اساس نیاز، یکی از ورژن ها را نصب کنیم.

### package.json
برای شروع باید ابتدا فایل `package.json` را بسازیم ک توسط دستور زیر می توان انجام داد:
```shell
npm init
```
حال باید ب سوالات جواب بدهیم و سپس فایل ما ساخته می شود.

### start
برای استارت زدن یک `server` فقط کافیست ب صورت زیر عمل کنیم :
```js title="app.js"
const http = require("http");

const server = http.createServer((req, res) => {
  console.log(req)
});

server.listen(3090, () => console.log("server is running..."));
```
اگر بخواهیم فایل را اجرا کنیم باید از دستور زیر استفاده کنیم:
```shell
node app.js
```
حال باید `localhost:3090` باز کنیم و در `terminal` ادیتور لاگ `req` را ببینیم.

## API's
### process
یکی از مهمترین `api` ها می باشد ک مانند `document` و `window` در وب می باشد و همچنین چون بر اساس `EventEmmiter` می باشد، پس می توان از رویداد ها هم در `process` استفاده کرد.

[node => process](https://nodejs.org/dist/latest-v16.x/docs/api/process.html)

در واقع تمامی اطلاعات مربوط ب سیستم ما در این آبجکت می باشد و بسیار مهم است مانند: ورژن نود، ورژن ویندوز، تمامی متغییر های محیطی(تمامی موارد مهم مانند پسورد دیتابیس ها و... در اینجا ذخیره می شوند.) `env` یا `kill` کردن یک پورت خاص یا خارج از شدن از نرم افزار یا دسترسی ب آرگومان های داده شده ب برنامه برای اجرا شدن (`node app.js`) یا نمایش `directory` و...
```text
process.pid ===> app id
process.arch ===> x86 or x64
process.version ===> node version
process.platform ===> win32
process.env ===> environment variables
process.exit() ===> close app
process.kill() ===> kill port
process.on() ===> events
process.argv ===> command with flags (npm start)
process.cwd() ===> directory
```
🎉 برای حل مشکل `intellisense` برای `node` باید از دستور زیر استفاده کنیم:
```shell
npm i @types/node
```
### OS & FS
#### FS
توسط `fs` ما می توانیم ب فایل های سیستم دسترسی داشته باشیم و چیزی درون آن ها بنویسیم یا تغییرشان بدهیم و ...

[node => fs](https://nodejs.org/dist/latest-v16.x/docs/api/fs.html)

خیلی از موارد `fs` دو حالت معمولی (`async`) و یک حالت `sync` دارد ک ب صورت پیش فرض نود `async` می باشد ک موارد سنگین را ب `event loop` میفرستد و سراغ پروسس بعدی می رود و مانند حالت `sync` صبر نمی کند تا پروسس تمام شود و سپس سراغ بعدی برود.

همچنین تمامی موارد `async` دارای `callback` هستند ک بعد از اجرا یک کار دیگر هم انجام بدهد.
```text
fs.writeFile() ===> async write file
fs.writeFileSync() ===> sync write file
fs.appendFile() ===> async append to a file
fs.appendFileSync() ===> async append to a file
fs.rename() ===> rename
fs.unlink() ===> async delete
fs.unlinkSync() ===> sync delete
```
#### OS
یک سری از موارد `process` را نیز برای ما انجام می دهد ولی در حالت کلی تمامی اطلاعات سیستم عامل درون این بخش می باشد.

[node => os](https://nodejs.org/dist/latest-v16.x/docs/api/os.html)
```text
os.platform()
os.arch()
os.userInfo() ===> user information like name, home directory, etc
os.release() ===> windows version
os.hostname() ===> host name
os.uptime() ===> computer uptime
```
